ALGOSONAR PROJECT - FILES AND DIRECTORIES EXPLANATION
=====================================================

This document explains the purpose of major directories and files in the algosonar Flutter project.

1. BUILD DIRECTORY (1,710 files)
================================
Purpose: Stores compiled/generated project output
- Location: /build/
- Contains:
  * Compiled Android APKs and native binaries
  * Generated intermediate files from the build process
  * Flutter compiled assets
  * Gradle cache and build artifacts
  * WebView Flutter compiled libraries
  * Native asset binaries
  * Build reports

Why we need it:
- Generated automatically during build process
- Contains compiled artifacts needed to run the app
- Stores temporary build files
- Safe to delete - will be regenerated on next build

2. ANDROID DIRECTORY (42 files)
================================
Purpose: Android-specific project configuration and native code
- Location: /android/
- Contains:
  * build.gradle.kts - Gradle build configuration
  * settings.gradle.kts - Gradle plugin and repository setup
  * gradle.properties - Gradle JVM and Android settings
  * gradlew & gradlew.bat - Gradle wrapper scripts
  * local.properties - Android SDK and Flutter SDK paths
  * app/ - Android application module
  * gradle/ - Gradle wrapper version config

Why we need it:
- Defines how to compile Flutter into Android APK
- Configures Android build system (Gradle)
- Stores Android-specific dependencies and settings
- Manages SDK versions and compilation targets
- Necessary for building Android app package

3. .DART_TOOL DIRECTORY (36 files)
===================================
Purpose: Dart/Flutter internal build cache and generated files
- Location: /.dart_tool/
- Contains:
  * Package dependency information
  * Dart kernel compilation cache
  * Generated build metadata
  * Package snapshot data
  * Flutter build configuration

Why we need it:
- Generated automatically by Flutter/Dart tooling
- Caches compiled Dart code for faster rebuilds
- Stores dependency resolution information
- Speeds up development builds significantly
- Safe to delete - will be regenerated

4. LIB DIRECTORY (7 files)
===========================
Purpose: Main application source code
- Location: /lib/
- Contains:
  * main.dart - Application entry point
  * screens/ - UI screen components
  * services/ - Business logic and API services
  * theme/ - App styling and theming

Why we need it:
- Contains all Dart/Flutter application code
- This is where you write features and UI
- Compiled into APK during build process
- Core of the application logic

5. .IDEA DIRECTORY (5 files)
=============================
Purpose: IDE (Android Studio/IntelliJ) project configuration
- Location: /.idea/
- Contains:
  * Project structure settings
  * IDE workspace preferences
  * Editor configurations
  * Build configuration for IDE

Why we need it:
- Required by Android Studio to recognize project
- Stores IDE-specific settings
- Enables code completion and debugging
- Configures run/debug profiles
- Safe to delete - will be regenerated by IDE

SUMMARY
=======
These directories work together to:
1. Store your source code (lib/)
2. Configure the build system (android/)
3. Cache dependencies and optimizations (.dart_tool/)
4. Generate compiled app (build/)
5. Integrate with development IDE (.idea/)

DEVELOPMENT WORKFLOW
====================
1. Developer writes code in lib/
2. Flutter build system uses android/ configuration
3. .dart_tool/ caches the compiled code
4. .idea/ provides IDE support during development
5. build/ stores final compiled artifacts

BUILD PROCESS
=============
Source Code (lib/) → Dart Compiler → .dart_tool/ cache → Android Gradle (android/) → APK Builder → build/app-release.apk

MAINTENANCE NOTES
=================
- build/, .dart_tool/ can be safely deleted (auto-regenerated)
- .idea/ can be regenerated by IDE
- NEVER delete: lib/, android/ (they contain essential code)
- Regular 'flutter clean' removes build artifacts
